Como usar este plano

  • Uma fonte principal (docs oficiais do Spring) + uma secundária (Baeldung, livro, vídeo).
  • Crie flashcards (ou lista de perguntas) de cada dia e revise em D+1, D+3, D+7, D+15.
  • Seu "superpoder" é o output: código rodando + testes passando + explicação clara.
  • Interleaving é obrigatório: misture temas para treinar diagnóstico e escolha de padrões.

Ritual diário (checklist mental)

  • Recuperação ativa: sem consultar, escreva o que lembra + 5 perguntas.
  • Aprender: leia/assista só o suficiente para corrigir falhas.
  • Construir: implementar uma feature (endpoint/service/repo/teste).
  • Interleaving: 2 micro-desafios misturando hoje + dias anteriores.
  • Feynman: explique em 10–15 linhas (ou áudio curto) como se ensinasse.

Repetição espaçada (regra operacional simples)

Todo conceito importante vira pergunta/flashcard e é revisado em D+1, D+3, D+7, D+15. Se você usar Anki, ele automatiza; se não, use uma lista no seu editor/Notion com essas datas.

O que vira flashcard

  • Definições ("o que é @Transactional?")
  • Porquês ("por que usar DTOs em vez de entidades na API?")
  • Como fazer ("como configurar JWT no Spring Security?")
  • Armadilhas ("quando lazy loading causa N+1?")

Formato de pergunta (melhor para retenção)

  • Explique: "O ciclo de vida de um bean Spring."
  • Decida: "Repository direto ou Service aqui? Por quê?"
  • Implemente: "Escreva um controller com tratamento de exceção sem olhar."
  • Depure: "Por que estou tendo LazyInitializationException?"
